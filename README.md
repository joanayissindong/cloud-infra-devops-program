### Structure en 8 phases

| Phase | Titre | Missions | Focus |
|---|---|---|---|
| 1 | Fondations Linux & Réseau | 01–05 | Linux, VPS, Nginx, Git, réseau |
| 2 | Windows Server & Active Directory | 06–09 | AD DS, GPO, PowerShell |
| 3 | M365, Entra, Intune & Windows Hybride | 10–15 | Microsoft 365, Entra ID, Intune, Hyper-V, Azure Arc |
| 4 | Docker, CI/CD & Automatisation | 16–20 | Docker, GitHub Actions, sécurité pipeline |
| 5 | Cloud Azure & AWS | 21–27 | Azure, AWS, Terraform, réseau hybride, haute disponibilité |
| 6 | Monitoring & Sécurité (SOC) | 28–30 | Prometheus, Grafana, Microsoft Sentinel |
| 7 | Kubernetes & Cloud-Native | 31–32 | k3s, AKS, RBAC, Ingress |
| 8 | DevOps Avancé & SRE | 33–38 | VMware, Okta, GitOps/ArgoCD, SAST/DAST, SRE |
| Final | Architecture d'entreprise | 39–40 | Dossier d'architecture, portfolio |

### L'univers du programme

Toutes les missions se déroulent dans un fil narratif continu plutôt que comme des cas isolés :
je démarre comme consultant chez **NexaTech Services** (ESN, Douala), avant d'être recruté en interne
par **FinCore Bank** — une banque panafricaine fictive — et son écosystème (FinCore Assurances,
FinCore Digital, Fondation FinCore). Chaque mission indique aussi *pourquoi* elle existe, certaines
comportent un imprévu à diagnostiquer en cours de route, et une soutenance a lieu toutes les 5 missions.

## 🗂️ Organisation du repo

```

├── README.md                                        → ce fichier
├── PROGRESS.md                                       → suivi de progression détaillé
├── Programme_Infrastructure_Cloud_40_Missions_v4.pdf  → programme complet

    ├── 01-serveur-linux-pme/
    ├── 02-vps-nginx/
    ├── 03-sauvegardes-automatisees/
    └── ...                                           → un dossier par mission terminée
```

Chaque dossier de mission contient, selon le cas : scripts, fichiers de configuration, schémas
d'architecture, procédure d'exploitation, et un README propre à la mission expliquant le contexte,
les choix techniques et les résultats.
